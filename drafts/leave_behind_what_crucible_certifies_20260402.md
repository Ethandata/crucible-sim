## The Problem We Solve

Enterprise AI agents fail in production — not because the models are bad, but because **reliability as a property of agentic systems has never been formally defined, measured, or certified.**

Every team evaluates differently. No results are portable. Security reviews have no standard to reference. Deployment decisions are made on gut feel and internal demos.

---

## What Crucible Tests

Crucible runs structured reliability evaluations across five dimensions:

| Dimension | What We Measure |
|---|---|
| **Tool Use Reliability** | Correct tool selection, parameter accuracy, error handling on unexpected returns |
| **Memory & State Management** | Context accuracy across multi-step tasks; stale vs. current state discrimination |
| **Recovery Behavior** | Failure detection, retry logic, safe halting, error non-compounding |
| **Scope Adherence** | Boundary compliance under edge-case and adversarial conditions |
| **Behavioral Consistency** | Output variance across repeated equivalent runs |
